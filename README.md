
Personal Trainer Landing Page
https://kax3per.github.io/PersonalTrainer/
============================

A modern, high-performance landing page built for a personal trainer brand.
Focused on clean design, strong UX, and real conversion flow.
------------

Overview
This project is a fully responsive fitness website designed to present a personal trainer, communicate value, and guide users through a structured transformation journey.
The layout follows a clear flow:
Hero → Trainer → Offer → Transformation → Pricing
Each section is built to serve a purpose — from capturing attention to building trust and driving action.


Key Features
 - UX & Structure
Logical section flow designed for conversion
Smooth scroll navigation with offset handling
Scroll-based active navigation (scroll spy)
-----------
 - UI Design
Modern, minimal aesthetic
Glassmorphism and soft shadows
Consistent spacing and typography system
Fully responsive (mobile-first approach)
-----------
 -  Interactivity
Flip card interaction (Offer section)
Drag-based transformation slider
Animated transitions and hover effects
Dynamic mobile navigation menu
-----------
  - Performance
Image preloading strategy
Lightweight architecture
Optimized animations (no heavy libraries)
-------------

Tech Stack
React – component-based architecture
Tailwind CSS – utility-first styling
JavaScript (ES6+) – logic and interactions
React Hooks – state and lifecycle management


The project is organized into clear layers:
components/ → reusable UI elements
sections/ → main page sections (Hero, Offer, etc.)
layout/ → wrappers and global structure
pages/ → page composition
Each section is built as an independent module, making the code scalable and easy to maintain.
Development Approach (AI + Manual Work)
This project was built using a modern workflow that combines manual development with AI-assisted tools.


AI is used to:
speed up development
improve code consistency
explore UI/UX solutions faster
assist in problem solving
However, all key decisions — including layout, structure, and interactions — are designed and implemented manually.
AI is a tool, not a replacement.
This approach allows for:
faster iteration
cleaner code
better overall quality
